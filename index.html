Analyzing Energy Consumption & Solar Potential Using SQL

This project explores energy consumption trends and solar energy potential using real-world data. The dataset contains 10-minute interval readings of power consumption across multiple zones, as well as solar radiation metrics such as General Diffuse Flows and Diffuse Flows. SQL was used for data cleaning, transformation, and analysis to identify key energy trends and optimize energy usage.

Skills Used:

  • Data Cleaning & Transformation
  • SQL Queries & Aggregations
  • Window Functions
  • Solar Energy Potential Modeling

Key Tasks Performed:

  • Processed Datetime Data: Standardized timestamps and extracted key time-based features for analysis.
  • Calculated Total Power Consumption: Aggregated energy usage across multiple zones to identify peak demand hours.
  • Analyzed Energy Demand Patterns: Grouped data by time periods (morning, evening, night) to detect consumption trends.
  • Computed Solar Potential: Normalized solar radiation values to assess the feasibility of solar energy generation.
  • Ranked Consumption Trends: Used window functions to determine the highest energy-consuming periods.
  • Identified Peak Demand Hours: Sorted and filtered data to highlight the highest energy usage periods.

By structuring and analyzing this dataset, valuable insights were uncovered regarding peak power demand, solar energy feasibility, and opportunities for energy optimization.

Cleaning and Transforming Nashville Housing Data Using SQL

This project focuses on cleaning and transforming raw real estate data from Nashville to ensure consistency and accuracy for analysis. The dataset contained missing values, inconsistent formats, and redundant information, requiring a structured approach to data cleaning using SQL.

Skills Used:

  • Data Cleaning
  • SQL Queries
  • Data Transformation
  • Standardization

Key Tasks Performed:

  • Standardized Date Formats: Converted sale dates into a consistent format for uniformity.
  • Handled Missing Data: Populated missing property addresses using Parcel IDs.
  • Split Address Columns: Separated full addresses into street, city, and state for better usability.
  • Standardized Categorical Data: Converted binary values in "Sold as Vacant" to "Yes/No".
  • Removed Duplicates: Identified and deleted duplicate records to maintain data integrity.
  • Dropped Unnecessary Columns: Eliminated redundant columns to optimize the dataset.

By implementing these transformations, the dataset is now structured, reliable, and ready for deeper analysis and visualization.

Covid-19
Data
Exploration

This project explores and analyzes global Covid-19 data, focusing on key metrics like cases, deaths, vaccinations, and their relationship with population statistics. It leverages various SQL techniques such as joins, common table expressions (CTEs), temporary tables, window functions, aggregate functions, data type conversions, and the creation of views to provide insights into the impact of the pandemic at a global and country level.

Skills Used:

  • Joins to combine data from Covid-19 death and vaccination tables.
  • Common Table Expressions (CTEs) to perform intermediate calculations.
  • Temporary tables for storing intermediate results.
  • Creation of views to store reusable query results for further analysis or visualization.

Key Tasks Performed:

  • Data Cleaning: Null values were handled for several columns in the Covid-19 deaths and vaccinations datasets by replacing empty strings with null values.
  • Basic Data Exploration: Retrieved essential columns (location, date, total cases, new cases, total deaths, population) to get a sense of the data for further analysis.
  • Statistical Analysis: Total Cases vs Total Deaths: Computed the death percentage to show the likelihood of death if infected. Total Cases vs Population: Calculated the percentage of the population infected by Covid-19. Countries with Highest Infection Rates and Death Counts: Identified countries with the highest infection rates and death counts, normalized by population. Global Numbers: Aggregated the total number of new cases and deaths globally to assess the death percentage globally.
  • Advanced Techniques: Utilized CTEs and temporary tables to store intermediate results for rolling vaccination counts and percentages. Created a view to encapsulate the vaccination data, making it reusable for future analysis or visualization.

Key Insights: The project provides a comprehensive view of Covid-19 data at a global and national level. It highlights trends in infection rates and vaccination progress, helping to identify areas of concern and success. The use of SQL’s advanced features (CTEs, window functions, views) ensures efficient and effective analysis of the large datasets involved.
